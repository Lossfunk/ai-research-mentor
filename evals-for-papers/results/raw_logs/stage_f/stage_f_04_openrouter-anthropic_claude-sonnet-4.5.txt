## **DETAILED VENUE SPECIFICATIONS**

### **1. NeurIPS 2025**

#### **Formatting Requirements**
- **Main paper**: 9 pages (including figures, tables)
- **References**: Unlimited (separate section)
- **Appendix**: Unlimited (reviewers not required to read)
- **Style file**: `neurips_2025.sty` (LaTeX required)
- **Font**: Times, 10pt minimum
- **Margins**: Strict - do not modify style file
- **Line numbers**: Required for submission
- **Figures**: Can be in main text or appendix

#### **Anonymization (STRICT)**
- ‚ùå No author names, affiliations, acknowledgments
- ‚ùå No self-citations that reveal identity ("our previous work [X]" ‚Üí "Prior work [X]")
- ‚ùå No URLs to personal/institutional websites
- ‚ùå Strip PDF metadata (author, institution)
- ‚úÖ Allowed: "Anonymous GitHub repository" links
- ‚úÖ Allowed: Cite your own work in third person
- **Violation = Desk rejection**

#### **Code & Data Artifacts**
- **Code release**: 
  - ‚úÖ Expected (NeurIPS Checklist asks explicitly)
  - ‚úÖ Can promise "upon acceptance" 
  - ‚úÖ Anonymous GitHub allowed during review (use [https://anonymous.4open.science/](https://anonymous.4open.science/))
  - License: Must specify (MIT, Apache 2.0 recommended)
- **Data release**:
  - ‚úÖ Expected if new dataset
  - ‚úÖ If using existing: cite and confirm license
  - ‚úÖ Processed splits should be shared
- **Models**:
  - ‚úÖ Pretrained model weights encouraged
  - ‚ö†Ô∏è Must discuss safety if releasing LLMs/sensitive models

#### **Required Checklists & Statements**
- ‚úÖ **NeurIPS Paper Checklist** (mandatory, in appendix):
  - Claims and evidence
  - Limitations
  - Theory assumptions
  - Experimental details (hyperparameters, seeds, compute)
  - Dataset documentation
  - Code availability
  - Broader impact
  - Safeguards (if applicable)
- ‚úÖ **Broader Impact Statement** (can be in main or appendix)
- ‚úÖ **Reproducibility**: Hyperparameters, seeds, hardware

#### **Supplementary Material**
- Format: Single PDF, unlimited pages
- Reviewers: Not required to read
- Contents: Proofs, additional experiments, dataset details, code snippets
- Submission: Same deadline as main paper

#### **For Your MIL Transfer Paper**
```
Page allocation strategy (9 pages):
- Abstract: 0.15 pages
- Introduction: 1 page
- Related Work: 1 page
- Methodology: 2 pages
- Experiments: 3.5 pages (tables, figures)
- Limitations: 0.35 pages
- Conclusion: 0.5 pages
- Broader Impact: 0.5 pages
= 9 pages total

Appendix (unlimited):
- NeurIPS Checklist: 2 pages
- Dataset details table: 0.5 pages
- Hyperparameter tables: 0.5 pages
- Additional ablations: 1-2 pages
- Failure case analysis: 1 page
- Computational resources: 0.5 pages
```

### **2. ICML 2025**

#### **Formatting Requirements**
- **Main paper**: 8 pages (including figures, tables, but NOT references)
- **References**: Unlimited (separate section)
- **Appendix**: Unlimited
- **Style file**: `icml2025.sty` (LaTeX required)
- **Font**: Times or similar, 10pt
- **Margins**: Strict - 1 inch all sides
- **Two-column**: No (single column)
- **Camera-ready**: +1 page allowed after acceptance

#### **Anonymization (STRICT)**
- ‚ùå Same as NeurIPS - very strict
- ‚ùå No acknowledgments of funding (reveals institution)
- ‚ùå No "we thank [specific lab]"
- ‚úÖ Can cite own work if properly anonymized
- **Special**: ICML has had issues with de-anonymization via code repositories - be careful

#### **Code & Data Artifacts**
- **Code release**:
  - ‚úÖ Strongly expected (Reproducibility Checklist)
  - ‚úÖ Anonymous repo allowed
  - ‚ö†Ô∏è Must work "out of the box" - reviewers may test
- **Data release**:
  - ‚úÖ Required if new dataset
  - ‚úÖ Must document existing datasets thoroughly
- **Reproducibility Checklist**: Mandatory
  - Includes: Algorithm pseudocode, hyperparameters, compute resources, statistical significance

#### **Required Checklists & Statements**
- ‚úÖ **Reproducibility Checklist** (in appendix)
- ‚úÖ **Societal Impact Statement** (0.5 page in main or appendix)
- ‚úÖ **Ethics Review**: Flagged during submission if human subjects

#### **Supplementary Material**
- Format: PDF, unlimited
- Can include: Code, data samples, videos
- Reviewers: Expected to review if referenced in main paper

### **3. ICLR 2026** (Deadline ~Sept 2025)

#### **Formatting Requirements**
- **Main paper**: No strict limit, but ~8-10 pages typical (excluding references)
- **References**: Unlimited
- **Appendix**: Unlimited
- **Style file**: `iclr2026_conference.sty`
- **Font**: 10pt
- **Flexibility**: More lenient than NeurIPS/ICML

#### **Anonymization (STRICT)**
- ‚ùå Same strict rules as NeurIPS/ICML
- ‚ö†Ô∏è **Special**: ICLR uses OpenReview - submissions are publicly visible (anonymously)
- ‚ö†Ô∏è Cannot post preprint to arXiv during review period (de-anonymization)
- ‚úÖ Can update submission during discussion phase

#### **Code & Data Artifacts**
- **Code release**:
  - ‚úÖ Strongly encouraged

  - ‚úÖ Reviewers often check code during review
- **Data release**:
  - ‚úÖ Expected for reproducibility
  - ‚úÖ Can be promised "upon acceptance"
- **OpenReview platform**: All supplementary materials uploaded there

#### **Required Checklists & Statements**
- ‚úÖ **Ethics Statement** (if applicable)
- ‚úÖ **Reproducibility Statement** (mandatory)
  - Must include: Code availability, data availability, hyperparameters, compute
- ‚úÖ **Limitations**: Expected in main paper

- Format: Uploaded to OpenReview (PDF, zip files)
- Size: Reasonable (no hard limit, but <100MB recommended)
- Reviewers: Expected to review

#### **Unique Aspects**
- **Open review**: Reviews are public (anonymously)
- **Author response**: Can respond to reviewers publicly
- **Discussion phase**: Reviewers and authors discuss openly
- **Community comments**: Anyone can comment (if enabled)
