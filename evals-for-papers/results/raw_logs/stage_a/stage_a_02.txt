## Initial Guidance (while you answer)

**Intuition**: Climate ML is vast, but beginners succeed by picking problems where (a) data is accessible, (b) you can validate results quickly (days, not months), and (c) the climate phenomenon is interpretable enough that you learn domain knowledge as you go. Starting with prediction or emulation tasks beats trying to build full climate models.

**Why this is principled**: Successful early-career projects balance tractability with learning value. The literature shows three beginner-friendly entry points: **(1) Emulation**—using ML to approximate expensive climate simulations [P2, P8], **(2) Prediction of specific events**—like wildfires [P1] or sea-level rise [P6], and **(3) Robustness testing**—checking if models generalize under climate change [P3]. All three let you work with public datasets, get feedback in weeks, and contribute meaningfully.

### Three Concrete Starting Paths

**Path 1: Wildfire Risk Prediction (Tabular ML + Explainability)**  
Replicate core experiments from [P1] using publicly available lightning and climate data to predict wildfire occurrence. Start with a simple random forest or gradient boosting model on historical data (1990–2010), validate on held-out years (2011–2015), then add SHAP explainability to identify which climate variables (temperature, precipitation, wind) matter most. Expected outcome: you'll achieve moderate AUC (0.65–0.75) and learn how climate features interact. Follow-up: test whether your model degrades on recent years (2016–2023) to explore nonstationarity [P3].

**Path 2: Precipitation Downscaling (Lightweight Emulation)**  
Build a simple convolutional neural network to "downscale" coarse climate model precipitation outputs to finer spatial resolution, inspired by [P2]. Use publicly available ERA5 reanalysis data (coarse) and gauge/satellite observations (fine) for a single region (e.g., California). Train a U-Net-style model to predict high-res precipitation from low-res inputs, evaluate with RMSE and spatial correlation, and visualize where the model struggles (e.g., mountainous regions). This teaches you about spatial ML and climate data quirks. Follow-up: test transferability to a different region [P8].

**Path 3: Out-of-Distribution Robustness Analysis (Model Auditing)**  
Pick an existing climate ML model from a recent paper with public code (e.g., temperature forecasting), reproduce its in-distribution performance, then systematically test it on "future-like" conditions by training on pre-2000 data and testing on post-2015 data [P3]. Measure performance degradation, identify which variables cause failures (using gradient-based attribution), and propose one architectural fix (e.g., adding trend features). This is lower compute, teaches critical thinking about generalization, and addresses a key open problem [P4, P5].
