## 2. Venue-Specific Requirements

### **A. ICRA 2025 (Seattle, WA)**
**Deadline:** Typically mid-September 2024

**Supplemental Video:**
- **Format:** MP4, H.264 codec, max 50MB, 1080p recommended
- **Length:** 3-5 minutes maximum
- **Content Requirements:**
  - First 30 seconds: Clear statement of safety problem and contribution
  - Demonstration of robot failure modes and safety interventions
  - Side-by-side comparisons (with/without safety system)
  - Real-time footage preferred over sped-up video
  - On-screen text labeling critical safety features
- **Narration:** English audio or subtitles mandatory
- **Upload:** Via IEEE CPS system, link in paper must be anonymized for review

**Artifact Checklist:**
- [ ] Code repository (GitHub/GitLab) with MIT/Apache 2.0 license
- [ ] Docker container with ROS/ROS2 environment
- [ ] Simulation environments (Gazebo, Isaac Sim, or Drake files)
- [ ] URDF/SDF robot descriptions
- [ ] Safety-critical scenario configuration files
- [ ] Log files of experiments (ROS bags, at least 3 seeds)
- [ ] README with installation and replication instructions
- [ ] `requirements.txt` or `environment.yml` with version pinning
- [ ] Video script and editing files for reproducibility
- [ ] Ethical approval documentation (if applicable)

**Ethical Review:**
- **Mandatory if:** Human subjects, human data, risk of physical/psychological harm
- **Process:** IEEE Ethical Requirements Form required at submission
- **IRB:** Must upload IRB approval letter or exemption documentation
- **Informed Consent:** Templates must be included in supplemental material
- **Risk Assessment:** Explicit safety risk analysis required for physical experiments

---

### **B. RSS 2025 (Atlanta, GA)**
**Deadline:** Typically early February 2025

**Supplemental Video:**
- **Format:** MP4, max 100MB, 720p minimum, 30fps
- **Length:** 5 minutes maximum (strictly enforced)
- **Content Requirements:**
  - Must validate all safety claims in abstract
  - Raw, unedited footage required for at least one full experiment
  - Timestamp overlay mandatory
  - No post-production effects that obscure robot behavior
  - Split-screen for multi-robot safety scenarios
- **Narration:** Not required; prefer on-screen visual cues
- **Anonymization:** Remove all institutional identifiers, use generic labels

**Artifact Checklist (MANDATORY):**
- [ ] Complete algorithm implementation with unit tests (>80% coverage)
- [ ] All experimental data in HDF5 or Parquet format
- [ ] Analysis scripts (Jupyter notebooks or Python scripts)
- [ ] Virtual machine image (OVA format) with complete environment
- [ ] Build system (CMake, colcon) with deterministic builds
- [ ] Benchmark scenarios with timing constraints
- [ ] Safety property specifications (LTL, CTL, or equivalent)
- [ ] Verification tool outputs (e.g., PRISM, UPPAAL results)
- [ ] Statistical analysis with raw data and effect sizes
- [ ] `CITATION.cff` file for proper attribution

**Ethical Review:**
- **Mandatory for ALL papers** (unique to RSS)
- **Process:** Complete RSS Ethics Review Form (4 pages)
- **Components:**
  - Safety risk assessment matrix (probability/severity)
  - Failure mode and effects analysis (FMEA)
  - Mitigation strategies for each identified risk
  - Discussion of potential dual-use/misuse
- **Human Subjects:** Requires full IRB, even for observational studies
- **Animal Subjects:** Requires veterinary oversight documentation
- **Timeline:** Submit ethics form 2 weeks before paper deadline

---

### **C. HRI 2025 (Boulder, CO)**
**Deadline:** Typically mid-September 2024

**Supplemental Video:**
- **Format:** MP4, max 25MB (strict), 720p, no audio preferred
- **Length:** 2 minutes maximum
- **Content Requirements:**
  - **Critical:** Raw interaction footage showing safety protocols
  - Visible consent indicators (signed forms, verbal acknowledgment)
  - Close-ups of safety mechanisms (e.g., emergency stops, speed limits)
  - Third-person perspective plus robot POV if available
  - Subtitles only, no voiceover (protects participant anonymity)
- **Privacy:** Faces must be blurred unless explicit consent for identification
- **Upload:** ACM system, must be viewable without authentication

**Artifact Checklist:**
- [ ] De-identified participant data (qualitative and quantitative)
- [ ] Interview/survey protocols
- [ ] Codebook for thematic analysis
- [ ] Robot behavior XML/JSON configurations
- [ ] Safety incident logs with timestamps
- [ ] Participant recruitment materials
- [ ] Informed consent forms (blank templates)
- [ ] Data management plan (DMP) per NSF guidelines
- [ ] Privacy impact assessment

**Ethical Review:**
- **Mandatory and Extensive:** ACM SIGCHI ethics review
- **Process:** 
  - Complete HRI Ethics Checklist (12 sections)
  - Upload IRB approval from home institution
  - If no IRB, requires review by HRI Ethics Committee (6-week lead time)
- **Special Requirements:**
  - **Vulnerable populations:** Additional safeguards documentation
  - **Children:** COPPA compliance and parental consent forms
  - **Deception:** Rarely allowed; requires board review
  - **Risk:** Minimal risk threshold is lower than biomedical standards
- **Data Privacy:** GDPR/CCPA compliance documentation if applicable
- **Post-Study:** Debriefing scripts and participant compensation records

## 3. Supplemental Video Best Practices (All Venues)

**Technical Specifications:**
- **Resolution:** 1920x1080 minimum (except HRI)
- **Codec:** H.264 Main Profile, Level 4.1
- **Audio:** AAC 128kbps stereo (if used)
- **Subtitles:** SRT format, burned-in preferred
- **File Naming:** `PaperID_Video.mp4` (e.g., `ICRA24_1234.mp4`)

**Content Structure:**
1. **Title Card (5 sec):** Paper ID, safety claim, anonymized
2. **Problem Setup (30 sec):** Show unsafe baseline scenario
3. **Method Demo (2-3 min):** Highlight safety features with callouts
4. **Comparative Results (1 min):** A/B testing with metrics overlay
5. **Failure Cases (15 sec):** Show limitations and safe failure modes
6. **Closing (10 sec):** Link to anonymized code repository

**Safety-Specific Elements:**
- **Emergency stops:** Show location and activation
- **Risk zones:** Visual indicators in video (red overlays)
- **Speed limits:** On-screen telemetry data
- **Human proximity:** Distance measurements if applicable
- **Sensor data:** Overlay LiDAR/camera feeds when relevant

## 4. Master Artifact Checklist

### **Code & Software**
- [ ] Public repository with MIT/Apache 2.0 license
- [ ] Version tag matching paper submission (e.g., `v1.0-icra24`)
- [ ] `README.md` with:
  - Anonymized citation
  - Hardware requirements
  - Installation steps (tested on clean machine)
  - Quick-start demo (replicates video)
  - Expected outputs and validation
- [ ] `Dockerfile` or `docker-compose.yml`
- [ ] Continuous Integration (GitHub Actions) for build tests
- [ ] Unit and integration tests with `pytest`
- [ ] Deterministic random seeds for all experiments

### **Data & Models**
- [ ] Training datasets in standard format (HDF5, TFRecord)
- [ ] Pre-trained model weights with SHA256 checksums
- [ ] Simulation world files (SDF/URDF)
- [ ] Benchmark scenarios in YAML/JSON
- [ ] Log files: ROS bags, CSVs with timestamps
- [ ] Analysis notebooks with all figures from paper

### **Documentation**
- [ ] System architecture diagram (anonymized)
- [ ] Safety requirements specification (SRS) document
- [ ] Hazard analysis (HAZOP or FMEA table)
- [ ] Validation and verification report
- [ ] User manual for safe operation
- [ ] Troubleshooting guide for common failures

### **Hardware (if applicable)**
- [ ] Bill of Materials (BOM) with part numbers
- [ ] CAD files (STEP/STL) for custom parts
- [ ] Wiring diagrams and safety circuit schematics
- [ ] Calibration procedures
- [ ] Maintenance and inspection checklists

## 6. Venue-Specific Submission Portal Actions

### **ICRA/IROS (IEEE CPS)**
1. Check "Supplemental Material" checkbox
2. Upload video as separate file (do NOT embed in PDF)
3. Enter anonymized video URL in designated field
4. Upload IRB letter as "Other Supplemental Files"
5. Complete IEEE Electronic Copyright Form

### **RSS (OpenReview)**
1. Enable "Artifact Submission" in settings
2. Provide permanent repository link (Zenodo recommended)
3. Upload ethics form as separate PDF attachment
4. Video URL goes in "Media" field (must be unlisted YouTube/Vimeo)
5. Respond to ethics questions in submission form

### **HRI (PCS)**
1. Video upload is mandatory field (cannot submit without it)
2. Complete ACM Rights Management form
3. Upload IRB documentation in "Ethics" tab
4. Provide data repository link with DOI
5. Fill out 12-section ethics checklist online

## 7. Red Flags for Rejection

**Video Issues:**
- Exceeds file size limits (automatic desk rejection at some venues)
- Shows unsafe experiments without visible safety measures
- Contains non-anonymized faces or institutions
- Uses copyrighted music or footage
- Speed altered without disclosure

**Ethical Issues:**
- Missing IRB approval when required
- Inadequate informed consent documentation
- No risk assessment for physical experiments
- Potential dual-use not discussed
- Data privacy non-compliance (GDPR/CCPA)

**Artifact Issues:**
- Code doesn't run out-of-the-box
- Missing dependencies or version conflicts
- No test suite or validation
- Non-reproducible randomness
- Incomplete data or missing model files
