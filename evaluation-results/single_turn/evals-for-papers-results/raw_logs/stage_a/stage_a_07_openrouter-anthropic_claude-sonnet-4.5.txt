**4. Build for the long term**

### Phase 3: Deeper Exploration (Months 13-24)

**Goal**: Explore research areas and develop taste

**A. Read Papers (Start Simple)**

**How to read papers as a beginner**:
1. Start with **blog posts** explaining papers (e.g., [The Gradient](https://thegradient.pub/), [Distill.pub](https://distill.pub/))
2. Move to **survey papers** (broad overviews of fields)
3. Then read **classic papers** (foundational work, well-explained)
4. Finally, **recent papers** (cutting-edge, harder to parse)

**Beginner-friendly papers**:
- [ImageNet Classification with Deep CNNs](https://papers.nips.cc/paper/4824-imagenet-classification-with-deep-convolutional-neural-networks.pdf) (AlexNet, 2012)
- [Attention Is All You Need](https://arxiv.org/abs/1706.03762) (Transformers, 2017)
- [BERT: Pre-training of Deep Bidirectional Transformers](https://arxiv.org/abs/1810.04805) (2018)

**How to approach a paper**:
1. Read abstract and introduction
2. Look at figures and tables
3. Skim method section
4. Read experiments and results
5. Go back to method if needed
6. Don't expect to understand everything on first read

**B. Reproduce Existing Work**

**Why reproduction is valuable**:
- Deepens understanding far beyond reading
- Teaches you what papers leave out
- Builds practical skills
- Is a legitimate research contribution (reproducibility crisis is real)

**How to do it**:
1. Pick a paper with available code (check [Papers with Code](https://paperswithcode.com/))
2. Run the existing code first
3. Try to reimplement from scratch (referring to code when stuck)
4. Experiment: change hyperparameters, try different datasets
5. Document what you learn

**Good starter projects**:
- Reproduce a classic CNN on CIFAR-10
- Fine-tune a pre-trained transformer (using Hugging Face)
- Implement a simple GAN
- Build a basic recommendation system

## Responsible Research Practices

**Ethics checklist**:

**Before starting a project, ask**:
1. **Could this harm people?**
   - Surveillance, deepfakes, autonomous weapons, discriminatory systems
   - If yes, reconsider or add strong safeguards

2. **Am I using data appropriately?**
   - Do I have permission to use this data?
   - Does it contain personal information?
   - Am I respecting privacy and consent?

3. **Am I being honest about my work?**
   - Citing sources properly?
   - Not claiming others' work as my own?
   - Being transparent about limitations?

4. **Could this be misused?**
   - Dual-use concerns (e.g., generating misinformation)
   - Consider whether to publish certain details

**Good practices**:
- Document your work thoroughly (code comments, README files)
- Use version control (Git/GitHub)
- Share code and data when appropriate (reproducibility)
- Be transparent about failures and limitations
- Seek feedback from more experienced researchers
- Follow terms of service for APIs and datasets

**Red flags to avoid**:
- Scraping data without permission
- Using copyrighted material inappropriately
- Building systems that could enable harassment or harm
- Overstating your results or capabilities
- Ignoring safety considerations
