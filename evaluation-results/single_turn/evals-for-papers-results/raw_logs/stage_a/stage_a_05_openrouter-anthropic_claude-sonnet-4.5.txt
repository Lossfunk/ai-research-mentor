## Emergency Strategy (Assuming Master's Thesis)

Given the one-month constraint, here's a **survival framework**:

### Week 1: Scope & Setup (Days 1-7)

**Pick a project type that's feasible:**

**Option A: Reproduction + Extension Study** (RECOMMENDED for 1 month)
- Take a recent paper (published in last 1-2 years)
- Reproduce their main results
- Add ONE small extension (new dataset, ablation study, minor method variation)
- Frame as "validation and extension of [Paper X]"

**Option B: Comparative Study / Benchmark**
- Compare 3-4 existing methods on a specific task
- Use public implementations (don't code from scratch)
- Provide thorough empirical analysis
- Frame as "empirical comparison of approaches for [Task Y]"

**Option C: Application Paper**
- Apply existing ML method to a new domain/dataset
- Focus on domain-specific insights
- Less novelty required if application is interesting

**Option D: Survey/Literature Review** (if allowed)
- Comprehensive review of a narrow subfield
- Organize existing work, identify gaps
- Requires no experiments but extensive reading

### Week 2: Execution (Days 8-14)

**If experimental (Options A-C):**

**Days 8-10: Get code running**
- Use existing implementations (GitHub, Papers with Code)
- Get baseline results on your chosen dataset
- Don't write from scratch—adapt existing code

**Days 11-14: Run experiments**
- Focus on ONE clear research question
- Run your main experiments
- Start logging results in tables/figures

**If survey (Option D):**
- Read 30-50 papers (skim most, read 10-15 deeply)
- Create taxonomy or framework for organizing the field
- Identify 3-5 key themes or open problems

### Week 3: Analysis & Writing (Days 15-21)

**Days 15-17: Complete experiments/reading**
- Finish all experimental runs
- Create all figures and tables
- OR: Complete literature review

**Days 18-21: Write draft**
- Introduction (2-3 pages)
- Related work (3-5 pages)
- Method/Approach (3-5 pages)
- Experiments/Analysis (5-10 pages)
- Conclusion (1-2 pages)

### Week 4: Polish & Submit (Days 22-30)

**Days 22-25: Revision**
- Get feedback from advisor
- Revise based on comments
- Check all citations

**Days 26-28: Final polish**
- Proofread thoroughly
- Format according to requirements
- Prepare any required presentation

**Days 29-30: Buffer for emergencies**

## Red Flags to Avoid

**DON'T:**
- ❌ Pick something requiring novel architecture design
- ❌ Choose a problem requiring large-scale data collection
- ❌ Attempt to train large models from scratch
- ❌ Select a topic with no existing code/baselines
- ❌ Try to solve an open problem in the field
- ❌ Pick multiple disconnected ideas

**DO:**
- ✅ Build on existing work heavily
- ✅ Use pre-trained models and public datasets
- ✅ Focus on ONE clear, narrow question
- ✅ Leverage existing implementations
- ✅ Set realistic expectations with your advisor
- ✅ Have a backup plan if experiments fail
