### B. Self-Citation Anonymization

#### Rule: You CAN cite your own work, but must anonymize references to it

**Citation Patterns:**

```latex
% ‚ùå WRONG (reveals identity)
Building on our previous work [Smith et al. 2023], we extend...
In [5], we showed that...
As we demonstrated in Smith et al. [12]...

% ‚úÖ CORRECT (anonymized)
Building on previous work [5], this paper extends...
Prior work [5] showed that...
Smith et al. [5] demonstrated...

% In references:
% ‚ùå WRONG
\bibitem{smith2023}
Jane Smith, John Doe. 
``Our Previous Paper.''
\emph{ACL 2023}.

% ‚úÖ CORRECT (same citation, neutral language)
\bibitem{smith2023}
Jane Smith, John Doe. 
``Previous Paper Title.''
\emph{ACL 2023}.
```

**Key principle**: Cite your work in third person, don't use possessive language ("our", "we previously")

#### Handling Unpublished/Concurrent Work

```latex
% ‚ùå WRONG
We build on our concurrent submission to NeurIPS [Anonymous, 2024]

% ‚úÖ CORRECT (if allowed by venue)
We build on concurrent work [6]. 
% In references:
[6] Anonymous. Under review. Details omitted for double-blind review.

% ‚úÖ BETTER (if venue allows)
We build on recent work on [topic]. Full citation omitted for anonymity.
```

**Venue-specific policies:**
- **ACL/EMNLP**: Can cite own work, must anonymize discussion
- **NeurIPS/ICML**: Can cite own work, must not reveal identity
- **CHI/CSCW**: Can cite own work, use neutral language
- **UIST/IUI**: Same as CHI

### B. Data Release Strategies

#### Human Evaluation Data: Privacy Considerations

**What you CAN release:**
- [ ] Aggregated statistics (means, medians, counts)
- [ ] Anonymized ratings/scores (no demographic links)
- [ ] De-identified qualitative codes
- [ ] Synthetic data matching real data distributions
- [ ] Task completion times, error rates (aggregated)

**What you CANNOT release:**
- [ ] ‚ùå Raw transcripts with identifying information
- [ ] ‚ùå Video/audio recordings (even with consent)
- [ ] ‚ùå Demographic data linked to individual responses
- [ ] ‚ùå Free-text responses that may identify participants
- [ ] ‚ùå IP addresses, timestamps, or other metadata

**Anonymization strategies:**

1. **K-anonymity**: Ensure each participant is indistinguishable from at least k-1 others
   ```python
   # Example: Generalize ages to ranges
   # Before: [23, 24, 25, 45, 46, 67]
   # After: [20-30, 20-30, 20-30, 40-50, 40-50, 60-70]
   ```

2. **Aggregation**: Report only group-level statistics
   ```latex
   % ‚ùå WRONG (individual-level)
   P1 rated the system 4/5, P2 rated 5/5, P3 rated 3/5...
   
   % ‚úÖ CORRECT (aggregated)
   Mean rating: 4.2/5 (SD=0.8, N=50)
   ```

3. **Suppression**: Remove rare/identifying attributes
   ```latex
   % ‚ùå WRONG (may identify individual)
   One participant, a 72-year-old Tibetan-speaking female 
   professor, noted...
   
   % ‚úÖ CORRECT (suppressed)
   An older participant with relevant expertise noted...
   ```

4. **Perturbation**: Add noise to preserve privacy
   ```python
   # Add Laplace noise to counts
   import numpy as np
   true_count = 47
   noisy_count = true_count + np.random.laplace(0, 1)
   # Report: "Approximately 47 participants..."
   ```
