**Annotation example (Python + OpenCV):**
```python
import cv2
import numpy as np

def annotate_safety_video(input_video, output_video, safety_data):
    """
    Overlay safety metrics on robot video
    
    Args:
        input_video: Path to raw robot footage
        output_video: Path to save annotated video
        safety_data: CSV with [frame, distance_to_obstacle, force, ...]
    """
    cap = cv2.VideoCapture(input_video)
    fps = int(cap.get(cv2.CAP_PROP_FPS))
    width = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
    height = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))
    
    fourcc = cv2.VideoWriter_fourcc(*'mp4v')
    out = cv2.VideoWriter(output_video, fourcc, fps, (width, height))
    
    frame_idx = 0
    while cap.isOpened():
        ret, frame = cap.read()
        if not ret:
            break
        
        # Get safety metrics for this frame
        distance = safety_data[frame_idx]['distance']
        force = safety_data[frame_idx]['force']
        
        # Overlay text
        cv2.putText(frame, f"Distance: {distance:.2f}m", 
                    (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 
                    1, (0, 255, 0), 2)
        cv2.putText(frame, f"Force: {force:.2f}N", 
                    (10, 70), cv2.FONT_HERSHEY_SIMPLEX, 
                    1, (0, 255, 0), 2)
        
        # Draw safety zone (example: circle around robot)
        if distance < 0.5:  # Safety threshold
            color = (0, 0, 255)  # Red = unsafe
        else:
            color = (0, 255, 0)  # Green = safe
        cv2.circle(frame, (width//2, height//2), 100, color, 3)
        
        out.write(frame)
        frame_idx += 1
    
    cap.release()
    out.release()

# Usage:
# annotate_safety_video('raw_robot.mp4', 'annotated.mp4', safety_metrics.csv)
```

## Safety Constraints

Our method enforces the following safety constraints:

1. **Collision avoidance**: Maintain distance > 0.1m from obstacles
2. **Joint limits**: Stay within manufacturer-specified limits
3. **Velocity limits**: Max joint velocity < 2.0 rad/s
4. **Force limits**: Contact force < 10N (for safe human interaction)

Constraints are defined in `config/safety_config.yaml`:

```yaml
safety_constraints:
  collision_avoidance:
    min_distance: 0.1  # meters
    buffer: 0.05       # safety margin
  
  joint_limits:
    use_manufacturer_limits: true
    safety_margin: 0.1  # radians
  
  velocity_limits:
    max_joint_velocity: 2.0  # rad/s
    max_end_effector_velocity: 0.5  # m/s
  
  force_limits:
    max_contact_force: 10.0  # Newtons
    max_torque: 5.0  # Nm
```

## Safety Protocol

**Before every experiment:**
1. Clear workspace of unnecessary objects
2. Verify emergency stop button works
3. Test safety constraints in dry-run mode
4. Have observer present
5. Start with slow motion (0.1√ó speed)

**During experiment:**
1. Monitor safety metrics in real-time
2. Keep hand near emergency stop
3. Maintain safe distance (>2m)
4. Watch for unexpected behavior

**After experiment:**
1. Return robot to home position
2. Review safety logs for violations
3. Document any issues

Full protocol: `docs/safety_protocol.md`

## Troubleshooting

**Robot not responding:**
- Check ROS connection: `rostopic list`
- Verify robot is not in error state
- Restart robot controller

**Safety violations in simulation:**
- Check constraint parameters in `config/safety_config.yaml`
- Verify environment setup
- Increase safety margins

**Real robot behaves differently than simulation:**
- Calibrate camera and force sensor
- Tune controller gains
- Check for mechanical issues (friction, backlash)

See `docs/troubleshooting.md` for more details.
